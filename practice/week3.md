[이번 주 목표]
3주차. 파이게임 실행. ai와 함께 코드 제작 및 수정. 키보드로 파란 원을 움직이게 하고, 쉬프트로 가속.

[AI에게 물어본 질문]
Pygame 학습 및 개발 과정 정리 (질문–답변–적용 결과 중심)

1. Pygame 기초 이해
   질문: Pygame 코드의 각 줄이 하는 역할과 필요성에 대해 질문함.
   답변: 초기화, 화면 생성, 반복문 구조, 그리기 및 화면 업데이트의 개념을 설명받음.
   적용 결과: 게임이 반복문을 기반으로 실행되며, 매 프레임마다 화면을 다시 그린다는 구조를 이해함.

2. 그래픽 출력 구현
   질문: 화면에 파란 원이 나타나야 한다고 질문함.
   답변: draw.circle과 display.flip의 역할을 설명받음.
   적용 결과: 화면에 도형을 출력하고, 이를 사용자에게 표시하는 과정에 성공함.

3. 개발 환경 설정
   질문: Pygame 설치 방법에 대해 질문함.
   답변: 패키지 설치 방법과 실행 환경 설정 방법을 안내받음.
   적용 결과: Pygame을 정상적으로 설치하고 실행 환경을 구축함.

4. 입력 처리 및 이동 구현
   질문: 방향키 및 WASD를 이용한 객체 이동 방법을 질문함.
   답변: 좌표 변수(x, y)와 키 입력 처리 방식을 설명받음.
   적용 결과: 키 입력에 따라 객체의 위치가 변경되는 이동 시스템을 구현함.

5. UI 요소 출력
   질문: 프레임(FPS)을 화면에 표시하는 방법을 질문함.
   답변: FPS 값을 계산하고 텍스트로 출력하는 방법을 안내받음.
   적용 결과: 게임 상태를 텍스트 형태로 화면에 표시할 수 있게 됨.

6. 도형 기반 캐릭터 구현
   질문: 이미지 없이 캐릭터를 표현할 수 있는지 질문함.
   답변: 기본 도형을 조합하여 캐릭터를 구성하는 방법을 설명받음.
   적용 결과: 도형을 이용해 캐릭터 형태를 직접 구성하는 방법을 이해함.

7. 게임 구조 확장
   질문: 간단한 게임 제작 방법에 대해 요청함.
   답변: 충돌, 점수, 상태 등의 요소를 포함한 게임 구조를 설명받음.
   적용 결과: 게임의 기본 구성 요소와 흐름을 이해함.

8. 사용자 인터페이스 및 상호작용
   질문: 텍스트 기반 상호작용 및 선택지 구현을 요청함.
   답변: 텍스트 출력과 선택지 처리 방식에 대해 설명받음.
   적용 결과: 사용자 입력에 따라 흐름이 변화하는 구조를 구현함.

9. 디버깅 및 오류 해결
   질문: 코드 실행 중 발생한 오류에 대해 질문함.
   답변: 오류 메시지 해석 및 수정 방법을 안내받음.
   적용 결과: 문법 오류를 스스로 수정할 수 있는 기초 능력을 갖춤.

10. 애니메이션 구현
    질문: 속도 변화 및 꿀렁거리는 애니메이션 구현을 요청함.
    답변: 타이머 변수와 도형 변형을 활용한 애니메이션 방법을 설명받음.
    적용 결과: 시간 기반으로 변화하는 애니메이션을 구현함.

11. 타이머 개념 이해
    질문: squish_timer 변수의 의미에 대해 질문함.
    답변: 애니메이션 지속 시간을 제어하는 프레임 기반 변수로 설명받음.
    적용 결과: 시간과 프레임을 활용한 상태 제어 개념을 이해함.

12. 충돌 및 물리 시스템 구현
    질문: 경계선, 충돌 판정, 튕김 효과 및 메시지 출력 기능을 요청함.
    답변: 위치 제한, 반대 방향 이동, 프레임 기반 이동 및 텍스트 출력 방법을 설명받음.
    적용 결과: 객체가 경계에 닿았을 때 자연스럽게 반응하는 충돌 시스템을 구현함.

13. 시각 효과 및 연출
    질문: 충돌 시 메시지가 일정 시간 표시되고 사라지는 효과를 요청함.
    답변: 타이머와 투명도, 파티클 방식으로 텍스트를 분산시키는 방법을 설명받음.
    적용 결과: 시간에 따라 변화하는 시각적 효과를 구현함.

종합 결과
기초적인 그래픽 출력에서 시작하여 입력 처리, 이동, 애니메이션, 충돌 판정, 시각 효과까지 단계적으로 구현하였다. 질문을 통해 개념을 이해하고, 이를 코드에 적용하는 과정을 반복함으로써 인터랙티브 프로그램 설계 및 구현 능력을 향상시켰다.

명령어의 명칭 중 헷갈리거나 이해가 되지 않는 것들로 인해 시행착오를 겪음
→ ai에게 질문하고, Diffchecker 에서 수정된 내용들을 복기해 코드가 원활하게 작동하도록 하였음.

[배운점]
ai를 활용하여 코딩 작업을 효율적이게 할 수 있고, 시간을 단축할 수 있다. 다만 모든 걸 ai한테 맡기는 것은 추후 버그나 수정을
해야 할 떄 문제가 생길 수 있다. 그러니 ai에게 코드에서 모르는 점을 물어보고 알아두거나, 커밋 메시지를 잘 기입 해 놓는 것이
좋다.
